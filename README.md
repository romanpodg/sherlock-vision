# 🕵️‍♂️ Sherlock Vision

**Sherlock Vision** – это интерактивный текстовый детективный симулятор для ВКонтакте, в котором пользователь берет на себя роль следователя и проходит расследование вымышленного преступления.

Игрок изучает материалы дела, осматривает место происшествия, допрашивает виртуальных свидетелей, анализирует улики, создает фоторобот подозреваемого и в финале формулирует собственную версию произошедшего. После этого система оценивает, насколько версия игрока совпадает с правильным решением дела.

Проект реализован на связке:

- **VK Bot API** – пользовательский интерфейс и взаимодействие с игроком;
- **YandexGPT** – диалоги со свидетелями, анализ улик и оценка версии игрока;
- **YandexART** – генерация фотороботов подозреваемых и визуальных материалов;
- **PostgreSQL** – хранение данных пользователей, прогресса расследования и результатов генерации.

> Все дела, персонажи, улики и изображения в проекте являются вымышленными. Sherlock Vision – это учебный игровой симулятор, а не реальный криминалистический инструмент.

---

## 📌 Назначение проекта

Цель проекта – создать интерактивную систему, которая демонстрирует применение генеративного искусственного интеллекта в игровом сценарии.

Sherlock Vision объединяет текстовый детективный квест, диалоговую модель и генерацию изображений. Пользователь не просто читает заранее написанный сюжет, а взаимодействует с ботом: задает вопросы свидетелям, получает ответы в свободной форме, анализирует найденные предметы и принимает решение на основе собранных данных.

Проект может использоваться как:

- учебный пример интеграции VK-бота с нейросетевыми API;
- демонстрация работы YandexGPT и YandexART в едином приложении;
- прототип интерактивного текстового квеста;
- основа для дальнейшего развития детективной игры с динамическими сценариями.

## 🌟 Основные возможности

### 🧩 Интерактивное расследование

Пользователь проходит дело поэтапно:

1. получает вводную информацию;
2. осматривает место происшествия;
3. изучает список свидетелей и улик;
4. задает вопросы свидетелям;
5. получает экспертный анализ улик;
6. создает фоторобот подозреваемого;
7. формулирует финальную версию;
8. получает оценку расследования.

### 💬 Допрос свидетелей через YandexGPT

Свидетели отвечают не заранее заготовленными фразами, а в свободной форме. Для каждого свидетеля в сценарии задаются:

- роль в деле;
- характер поведения;
- известные ему факты;
- факты, в которых он сомневается;
- информация, которой он не владеет.

YandexGPT получает системный промпт и контекст конкретного свидетеля, поэтому отвечает от его лица и не должен раскрывать информацию, которой свидетель не знает.

Пример:

```text
Детектив: Вы видели, кто выходил через служебный вход?

Охранник: Да, я заметил человека около 21:40. Лицо было видно плохо, но он был в темной куртке и нес небольшой черный кейс.
```

### 🔎 Анализ улик

Игрок может выбрать улику и получить комментарий виртуального криминалиста.

YandexGPT анализирует описание предмета, объясняет его возможное значение и подсказывает, на что стоит обратить внимание. При этом анализ строится на данных конкретного дела, а не на случайной генерации.

Пример улики:

```text
Обрывок темной ткани найден возле служебной двери.
Возможный вывод: ткань могла оторваться от одежды человека, который проходил через этот вход.
```

### 🧑‍🎨 Генерация фоторобота подозреваемого

Sherlock Vision может создать изображение подозреваемого по описанию внешности.

Описание может быть:

- заранее задано в сценарии дела;
- собрано из показаний свидетелей;
- уточнено пользователем вручную.

YandexGPT формирует структурированный промпт, а YandexART создает изображение в стилистике полицейского фоторобота или детективного портрета.

Пример признаков:

```text
Мужчина 30–40 лет, худощавое телосложение, узкое лицо, короткие светлые волосы, темная куртка с капюшоном, нейтральное выражение лица.
```

### 🖼 Визуализация вещественных доказательств

Кроме фотороботов, проект может использовать YandexART для визуализации улик:

- обрывок ткани;
- след обуви;
- поврежденный замок;
- черный кейс;
- фрагмент записи с камеры;
- место происшествия.

Это делает расследование более наглядным и усиливает атмосферу детективного квеста.

### ✅ Финальная оценка расследования

В конце дела игрок пишет свою версию:

- кто совершил преступление;
- каким способом;
- какой был мотив;
- какие улики это подтверждают.

YandexGPT выступает в роли старшего инспектора и сравнивает ответ пользователя с правильным решением дела.

Итоговая оценка включает:

- процент точности версии;
- что определено верно;
- какие факты были упущены;
- какие выводы ошибочны;
- раскрыто ли дело.

## 🧠 Роль искусственного интеллекта в проекте

В проекте YandexGPT и YandexART не заменяют игровую логику полностью. Основой остается сценарный движок, который контролирует:

- состояние пользователя;
- доступные действия;
- факты дела;
- список свидетелей;
- список улик;
- правильное решение.

Нейросети используются как вспомогательные интеллектуальные инструменты.

### YandexGPT используется для:

- генерации ответов свидетелей;
- анализа улик;
- структурирования описаний подозреваемого;
- подготовки промптов для YandexART;
- оценки финальной версии игрока;
- форматирования ответов в стиле детективного помощника.

### YandexART используется для:

- генерации фоторобота подозреваемого;
- визуализации вещественных доказательств;
- создания изображений места происшествия;
- художественной реконструкции отдельных сцен.

## 🧱 Описание основных модулей

### `app/`

Точка входа в приложение. Здесь запускается бот, инициализируются настройки, подключение к базе данных, VK Long Poll и фоновые задачи.

### `core/`

Содержит центральную логику управления ботом:

- маршрутизацию входящих сообщений;
- обработку команд;
- определение состояния пользователя;
- передачу управления нужному игровому модулю.

### `game/`

Игровой слой проекта.

Здесь находятся:

- модели детективного дела;
- загрузчик сценариев;
- движок расследования;
- логика свидетелей;
- логика анализа улик;
- генерация фотороботов;
- финальная оценка версии игрока.

### `vk_bot/`

Модуль взаимодействия с ВКонтакте.

Отвечает за:

- получение новых сообщений;
- обработку событий Long Poll;
- отправку сообщений;
- создание клавиатур;
- загрузку изображений в VK;
- отправку сгенерированных фотороботов и улик.

### `yandex_ai/`

Модуль интеграции с Yandex Cloud Foundation Models.

Содержит:

- клиент для YandexGPT;
- клиент для YandexART;
- системные промпты;
- функции сборки промптов;
- обработку ответов нейросетей.

### `database/`

Слой работы с базой данных.

Используется для хранения:

- пользователей;
- активных расследований;
- текущего состояния игрока;
- выбранных свидетелей и улик;
- истории сообщений;
- информации о сгенерированных изображениях.

### `workers/`

Фоновые задачи.
Главная задача – обработка генерации изображений, так как YandexART работает не мгновенно. Пользователь получает сообщение о начале генерации, а воркер проверяет статус операции и отправляет изображение после готовности.

## 🧭 Состояния пользователя

Бот использует систему состояний, чтобы понимать, как интерпретировать сообщение пользователя.

Например, если пользователь находится в состоянии `WITNESS_DIALOGUE`, его сообщение считается вопросом свидетелю. Если пользователь находится в состоянии `FINAL_VERSION`, сообщение считается итоговой версией расследования.

Основные состояния:

```text
START
MAIN_MENU
CASE_INTRO
INVESTIGATION_MENU
LOCATION_INSPECTION
WITNESS_SELECTION
WITNESS_DIALOGUE
EVIDENCE_LIST
EVIDENCE_ANALYSIS
PORTRAIT_GENERATION
IMAGE_GENERATION_WAITING
FINAL_VERSION
CASE_RESULT
```

## 🚀 Установка и запуск

### Требования

Перед запуском необходимо иметь:

- Python 3.10 или выше;
- PostgreSQL;
- Git;
- аккаунт и сообщество ВКонтакте;
- доступ к Yandex Cloud Foundation Models и активные API-ключи.

### 1. Клонирование репозитория

```bash
git clone https://github.com/romanpodg/sherlock-vision.git
cd sherlock-vision
```

### 2. Создание виртуального окружения

#### Linux / macOS

```bash
python3 -m venv venv
source venv/bin/activate
```

#### Windows

```bash
python -m venv venv
venv\Scripts\activate
```

### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 4. Настройка переменных окружения

Создайте файл `.env` на основе `.env.example`:

```bash
cp .env.example .env
```

Заполните переменные окружения:

```env
VK_TOKEN=
VK_GROUP_ID=

YANDEX_API_KEY=
YANDEX_FOLDER_ID=

POSTGRES_HOST=localhost
POSTGRES_PORT=5432
POSTGRES_USER=
POSTGRES_PASSWORD=
POSTGRES_DB=sherlock_vision
```

Названия переменных или их порядок могут отличаться, поэтому проверяйте актуальный список параметров в `.env.example`.

### 5. Создание базы данных PostgreSQL

Пример создания базы данных:

```bash
createdb sherlock_vision
```

Или через `psql`:

```sql
CREATE DATABASE sherlock_vision;
```

### 6. Применение миграций

```bash
alembic upgrade head
```

### 7. Запуск бота

```bash
python -m app.main
```

После запуска бот начинает слушать входящие сообщения ВКонтакте через Long Poll.

## ⚠️ Ограничения проекта

Sherlock Vision не предназначен для реального расследования преступлений.

Ограничения:

- изображения YandexART являются художественной реконструкцией;
- фоторобот не является достоверным изображением человека;
- ответы свидетелей ограничены сценарием дела;
- YandexGPT может ошибаться, поэтому факты дела должны контролироваться игровым движком;
- проект работает только при наличии доступа к VK API и Yandex Cloud API.
