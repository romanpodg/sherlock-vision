# Описание и контекст проекта «Sherlock Vision»

«Sherlock Vision» — это текстовый детективный квест, реализованный в виде чат-бота для ВКонтакте. Игрок выступает в роли детектива, расследующего различные преступления. Бот предоставляет интерактивное меню, позволяя изучать место преступления, допрашивать свидетелей, исследовать улики, составлять фотороботы и в конце сдавать дело руководству. Уникальность проекта заключается в глубокой интеграции с нейросетевыми технологиями Яндекса, которые "оживляют" игровой процесс.

## Архитектура и стек технологий

Проект построен по модульному принципу, отделяя логику VK от игровой логики и работы с БД:
- **Язык**: Python 3.10+ (Асинхронный подход)
- **База данных**: PostgreSQL + SQLAlchemy (Async) + Alembic
- **VK API**: Прямое взаимодействие через `aiohttp` с использованием Long Poll.
- **Нейросети**: Yandex Cloud Foundation Models (YandexGPT Lite, YandexART).
- **Валидация данных**: Pydantic и pydantic-settings.

## Игровой цикл (Game Loop)

1. **Знакомство с делом**: Пользователь получает вводную информацию из JSON-файла дела.
2. **Осмотр локации**: Доступно статичное описание места преступления.
3. **Опрос свидетелей (С интеграцией YandexGPT)**: Пользователь выбирает свидетеля и задает ему вопросы в свободной форме. YandexGPT получает системный промпт с описанием характера свидетеля, его роли и известных ему фактов. Свидетель отвечает от своего лица, не выдумывая лишнего.
4. **Изучение улик (С интеграцией YandexGPT)**: Пользователь осматривает улику и может задать вопрос виртуальному эксперту-криминалисту.
5. **Создание фоторобота (С интеграцией YandexART)**: Игрок может запросить портрет подозреваемого. Фоновый процесс формирует промпт на основе внешности из дела и генерирует уникальное изображение.
6. **Вынесение вердикта**: Детектив описывает свою версию произошедшего. YandexGPT, выступая в роли старшего инспектора, оценивает точность ответа, сравнивая его с зашитым в дело решением.

## Проделанная работа (Поэтапно)

**Этап 1: Каркас проекта и База данных**
- Создана базовая структура директорий (`app`, `core`, `database`, `game`, `vk_bot` и др.).
- Настроен `pydantic-settings` для чтения ключей из `.env`.
- Спроектирована схема БД (PostgreSQL): модели пользователя, сессии расследования, логов сообщений, сгенерированных картинок.

**Этап 2: Интеграция VK API и Главное меню**
- Написан асинхронный клиент для VK API и слушатель событий `VkLongPoll`.
- Реализована отправка сообщений и инлайн-клавиатур (`VkSender`, `VkKeyboards`).
- Создан модуль маршрутизации (`Router`), который реагирует на команды меню.

**Этап 3: База данных и Сессии**
- Реализованы репозитории (`UserRepository`, `InvestigationRepository`) для управления состоянием игрока через базу данных.
- Состояние игрока теперь хранится в БД, что позволяет продолжать игру после перезапуска.

**Этап 4: Логика дел и Сценарий**
- Создан тестовый файл дела (`case_001_gallery_ring.json`) со всеми нужными данными (улики, свидетелей, решение).
- Написаны Pydantic модели (`game/case_models.py`) для строгой валидации JSON.
- Написан загрузчик дел `CaseLoader`.
- Создан движок сценариев `ScenarioEngine`, который переводит игрока по различным состояниям внутри дела (`INVESTIGATION_MENU`, `WITNESS_SELECTION` и т.д.).

**Этап 5: Интеграция YandexGPT (Свидетели и Улики)**
- Написан REST клиент `YandexGPTClient`.
- Разработаны системные промпты `system_prompts.py` и генератор промптов `prompt_builder.py`.
- Созданы `WitnessEngine` и `EvidenceEngine` для связки действий пользователя, данных из дела и ответов ИИ.

**Этап 6: Интеграция YandexART (Фоторобот)**
- Разработан клиент `YandexARTClient` для отправки асинхронных задач по генерации изображений.
- Создан фоновый процесс `ImageGenerationWorker`, который поллит статус генерации в Яндексе, скачивает готовую картинку, загружает ее на сервер VK через созданный `VkUploader` и отправляет игроку.
- Написан `PortraitEngine`, стартующий процесс по описанию внешности подозреваемого из дела.

**Этап 7: Финальная оценка и MVP тестирование**
- Создан системный промпт старшего инспектора.
- Написан `FinalEvaluator`, который сравнивает финальную версию пользователя с решением из JSON файла с помощью YandexGPT и выдает итоговую оценку.
- Обновлен сценарий для закрытия дела и возврата в главное меню.

## Текущий статус
Разработка минимально жизнеспособного продукта (MVP) полностью завершена. Система работоспособна, все модули интегрированы друг с другом.
